TheLoom v0.28.0 · R13 专业稿件导入与章节编译

R13 首批把已有长稿直接接入写作工作台：TXT、Markdown、EPUB 与 DOCX 共用一套卷 / 章 / 场景预检和事务式导入；完成编辑后，可按章节选择并编译为 Markdown、纯文本或 Final Draft。桌面项目同时升级为“新建即落盘、可选仅文件夹存储”的本地优先流程。

## 多格式长稿导入

- **TXT / Markdown**：识别中英文卷章标题、三级 Markdown 标题、分隔线、frontmatter 标题 / 作者和代码围栏；支持 BOM、严格 UTF-8，以及 GBK / GB18030 / Big5 / Shift-JIS 中文旧编码回退
- **EPUB**：从 `container.xml` 定位 OPF，按 spine 阅读顺序提取 XHTML；`h1` 划分章节、`h2-h6` 划分场景，并跳过脚本、样式、导航及非线性附录
- **DOCX**：直接解析 `word/document.xml` 和核心属性，按 Title / Heading 1-3 还原项目、卷、章、场景层级，保留换行与制表符
- 四种格式统一进入导入预检，展示卷 / 章 / 场景数量、目录树和警告；确认后只新增文档与文件夹，不覆盖既有内容，整次导入可一步撤销

## 章节编译

- 按卷 / 章分组勾选参与编译的场景，支持整组选择和半选状态
- 实时显示选中文档数、字符数与预计输出大小
- 导出 Markdown、纯文本或 Final Draft `.fdx`；按 Navigator 顺序拼接，并可附带卷 / 章路径标题

## 桌面本地存储

- 桌面版新建项目默认先选择本地文件夹；取消后仍可明确选择创建仅保存在应用内的轻量项目
- 选到已有项目目录时可直接作为新槽位打开，不会静默覆盖
- 项目完整落盘后可选择清除浏览器镜像和未被其他槽位 / 快照引用的资源缓存，进入“仅文件夹存储”；后续编辑不再重建整份镜像
- 项目菜单保留稍后清理入口；解除绑定前自动恢复本地镜像，更换目录时可直接从原文件夹复制资源
