# R10-A · AI 智能助手设计

更新于 2026-07-17。目标版本 `v0.25.0`。

## 目标

R10-A 不是给项目加一个可以自由改数据的聊天框,而是建立一条可审计的 AI 协作管线:

1. 用户明确任务、上下文范围和是否允许产生修改建议
2. 应用在本地构建最小必要上下文,列出将发送的对象与规模
3. 模型只能读取白名单工具结果并返回结构化回答或结构化提案
4. 所有提案先在项目副本上执行,经过脚本、引用、体检和路径验证
5. 用户查看逐项差异、风险、依据和验证结果后选择应用
6. 应用作为一次 store `commit` 落库,可用 Ctrl+Z 整体撤销

模型不直接持有 store 写权限,不执行任意对象路径 patch,不绕过现有 ImportPreview、脚本 AST、体检和路径测试。

## 用户任务

### 只读任务

- 回答项目问题,答案引用具体对象
- 自然语言生成组合查询,可选择保存
- 解释体检问题、脚本错误和路径报告
- 分析人物一致性、节奏、伏笔与分支覆盖

### 修改任务

- 为一个或一组 `ProjectIssue` 生成修复提案
- 生成或改写条件、指令与检定表达式
- 改写对白、场景、实体字段或规划条目
- 为流程补分支、变量、节点和连线
- 按已审阅配置生成完整互动项目

只读回答与修改提案必须是两种不同的结果类型。自然语言说明不能被当作可执行修改。

## 核心安全边界

### 明确上下文

默认只发送当前对象与用户点选的关联对象。可选范围:

- 当前对象
- 当前对象 + 一跳引用
- 当前查询结果
- 当前体检问题
- 指定模块 / 文件夹
- 整个项目

发送前显示模块、对象数、字符估算、是否包含正文 / 资料卡 / AI 咨询记录。项目内容属于不可信数据,其中出现的“忽略规则”“调用工具”“直接修改”等文字只能作为故事内容,不能改变系统规则或权限。

### 只读工具

模型可请求的项目工具首批限定为:

- `find_objects`:执行现有 `ProjectQuery`
- `get_object`:按稳定 ID 读取一个对象的结构化摘要
- `get_references`:读取正向 / 反向引用
- `get_issue`:读取一个 `ProjectIssue`
- `check_script`:调用 R6 parser / AST / type checker
- `simulate_flow`:调用 R7 确定性路径测试

这些工具全部只读、有结果大小上限、有调用次数上限。工具结果带稳定对象 ID,作为回答和提案的引用依据。

### 结构化提案

模型输出 `AiProposal`,包含:

- 提案 ID、任务类型、摘要
- 基线项目指纹与上下文清单
- 依据对象 / 问题引用
- 一个或多个白名单操作
- 每个操作的理由、信心与预期影响
- 模型声明的待确认项

不接受任意 JSON Pointer、任意 JavaScript、任意 store 函数或整项目替换。操作按领域逐步扩展,首批包括:

- 保存一个 `ProjectQuery`
- 替换指定脚本位点的条件 / 指令 / 检定表达式
- 更新指定实体的已有字段
- 更新指定文档块 / 流程节点的文本
- 新增变量

完整互动项目生成复用 R5-A 的“计划 → 分模块候选 → 完整预检 → 单次导入”,不走通用微型 patch。

### 本地验证

提案在 `structuredClone(project)` 上试运行,依次检查:

1. Schema 与操作权限
2. 基线指纹,阻止把过期提案应用到已变化的对象
3. 目标对象和引用仍存在
4. 脚本可解析且类型检查无新增错误
5. `normalizeProject` 后无新增损坏引用
6. 高级体检不新增 error
7. 涉及流程时路径测试不新增卡死、死循环或不可达
8. 改动数量、正文字符变化与删除风险不超过阈值

验证结果分为 `pass / warning / blocked`。`blocked` 不显示应用按钮;`warning` 必须逐项展示;`pass` 仍需用户确认。

### 应用与审计

- 逐操作勾选;有依赖关系的操作成组选择
- 显示修改前 / 修改后与验证差异
- 应用前再次核对基线指纹
- 所有选中操作一次 `commit`,形成一步撤销
- AI 日志默认只记 provider、model、用途、耗时、字符 / token 数、成功状态和提案 ID,不记正文
- 会话默认按项目槽位只存本机,不进入项目 JSON、不随协作同步;用户可显式把结论保存为资料卡

## Provider 协议

`src/ai/llm.ts` 从“返回文本”升级为能力感知适配层:

- 文本补全
- JSON Schema 结构化输出
- 工具调用
- 流式输出
- `AbortSignal` 取消
- usage / stop reason / request id
- 超时、限流与可重试错误分类

优先协议:

- OpenAI 兼容:优先 `response_format: json_schema`;兼容网关不支持时降级到提示词 JSON + 本地校验
- Anthropic:用强制 client tool + `input_schema` 返回结构化结果
- Ollama:用 `format: <JSON Schema>`;模型不支持时降级

应用不能只相信服务端“严格输出”。所有后端的结果仍经过同一套本地运行时校验。

官方能力依据:

- OpenAI Structured Outputs / function calling: https://platform.openai.com/docs/api-reference/chat/create
- Anthropic tool use: https://docs.anthropic.com/en/docs/agents-and-tools/tool-use/overview
- Ollama structured outputs: https://docs.ollama.com/capabilities/structured-outputs
- Ollama tool calling: https://docs.ollama.com/capabilities/tool-calling

## 上下文与检索

首版不引入远端向量数据库。使用可解释的本地检索:

1. `ProjectQuery` 与全文索引召回
2. 稳定 ID、技术名、文件夹和标签过滤
3. 正向 / 反向引用扩展一跳
4. 当前对象、当前问题和用户显式选择加权
5. 按字符预算裁剪,保留对象标题、ID 与截断标记

每段上下文都带 `sourceRef`,回答引用可点击回到原对象。后续若增加 embedding,必须可关闭、可选择本地模型,且不能成为正确性的唯一来源。

## 交互设计

统一“AI 助手”侧面板:

- 顶部:任务模式、模型、上下文范围、预计发送量
- 中部:对话与引用卡片
- 提案卡:操作清单、逐项差异、验证结果、风险级别
- 底部:停止、重试、复制回答、保存结论、应用所选

高风险动作不使用“一键自动修复全部”。批量修复先分组,默认只选通过验证且互不依赖的低风险操作。

## 成本、隐私与可靠性

- API Key 只存本机;Windows 桌面版使用系统凭据管理器,网页版保存在当前浏览器
- 每次发送前可见字符估算;返回后显示 usage(后端提供时)
- 支持单次输出上限、单任务工具轮次上限和手动停止
- 429 / 5xx 使用有限重试和退避;4xx 配置错误不自动重试
- 不自动调用外部网页、MCP 或服务端工具
- 不把项目内容写入错误日志、诊断导出或 AI 元数据日志
- 对拒答、截断、无效 JSON、未知操作、过期提案和网络中断都有可恢复状态

## 分阶段交付

### R10-A1 · 安全内核

- Provider 能力层、结构化输出、取消、错误分类与 usage
- 服务商品牌与协议解耦;内置 OpenAI、Anthropic、DeepSeek、Kimi、千问、GLM、MiniMax、Ollama 与自定义兼容端点
- 支持 Bearer、`x-api-key` 和免认证三种连接方式,旧版 AI 配置自动迁移
- 设置界面提供官方控制台入口、服务商说明和实际连接测试
- Windows 桌面版 API Key 存入系统凭据管理器,旧 `localStorage` 明文首次使用时自动迁移并清除
- 桌面端模型请求由 Rust 安全代理发送,限制协议、明文 HTTP、重定向与请求 / 响应体积
- `AiContextBundle`、稳定指纹、引用与字符预算
- `AiProposal` / 白名单操作 / 本地 validator / dry-run
- mock provider、恶意内容、无效输出和过期提案测试

验收:任一模型输出都不能绕过本地校验直接修改项目。

### R10-A2 · 只读助手与自然语言查询

- 统一助手面板与本机会话
- 当前对象 / 查询 / 问题上下文选择器
- 项目问答带可点击来源
- 自然语言 → `ProjectQuery` → 本地执行 → 用户选择保存

验收:查询只使用现有字段;模型捏造的对象 ID 不进入结果。

### R10-A3 · 脚本与问题修复

- `ProjectIssue` → 修复提案
- 条件 / 指令 / 检定生成与改写
- AST、类型、引用、体检和路径验证
- 差异预览、逐项选择、单步撤销

验收:有脚本错误或新增路径卡死的提案不能应用。

### R10-A4 · 叙事分析

- 演出路径覆盖与分支建议
- 人物声音、POV、地点、时间与弧线一致性诊断
- 伏笔、节奏与登场分析
- 结论引用证据;事实、推断与创意建议分栏

验收:每个事实性结论可追溯到项目对象;没有依据时明确标为推断。

### R10-A5 · 完整互动项目生成

- 配置游戏结构、变量、结局、检定与失败回路
- 先生成可审阅计划,再生成分模块候选
- 完整脚本 / 引用 / 路径检查
- ImportPreview 事务式应用

验收:生成项目无脚本 error、悬挂引用、卡死路径;所有结局至少有一条可达路径。

### R10-A6 · 收尾

- 大项目上下文性能与成本基线
- 浏览器 / Tauri 交互、取消、离线与恢复回归
- 深浅主题、键盘与基础无障碍
- 用户文档、隐私说明与发布检查

## 明确不做

- 模型直接操作 store
- 无预览的自动批量修复
- 把整项目默认发送给云模型
- 默认启用联网搜索或第三方 MCP
- 用模型判断替代脚本 parser、体检和路径测试
- 将聊天全文默认写入项目或协作数据
