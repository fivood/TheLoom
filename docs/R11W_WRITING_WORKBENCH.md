# R11-W 完整写作工作台

状态：已随 `v0.26.0 / R11` 发布，作为 R11-2 与 R11-3 之间的插入批次。

R11-W 的目的不是增加另一份正文，而是让同一组 `Document.blocks` 同时服务沉浸写作、结构整理、连续稿、流程节点和 Obsidian 往返。

## 已完成

- 文档模块拆分为写作、结构、连续稿三种视图，增加隐藏 Navigator 与属性栏的专注模式
- 新建对象明确为场景；一个场景以 `Document.name` 命名，正文默认从 `paragraph` 块开始，不再重复创建空场景标题
- Enter 新段、Shift+Enter 块内换行、Alt+↑/↓ 移动、`/` 块类型菜单、新块插入到当前块之后并自动聚焦
- 常用正文、动作、对白、场景入口前置；低频写作块与逻辑块收进更多菜单
- 对白说话人支持输入搜索、键盘选择和就地创建角色
- 结构视图支持拖拽重排，并为正文 / 动作 / 对白设置 `none / beat / node` 流程角色
- 普通正文默认不生成流程节点；显式节拍正文与剧本块继续使用 `NarrativeUnit` 双向同步
- 文档首次生成流程后写入稳定关联；后续可直接打开，或预览节点 / 连线数量变化后确认重建结构
- 文件夹 Markdown 改为可读正文为权威内容，隐藏 `loom:block` 标记只保存稳定 ID、类型与引用；旧 `yaml loom-blocks` 继续兼容
- 无织机标记的外部 Markdown 按自然段导入为普通正文块
- 文档 Navigator 的卷 / 章文件夹映射为 `documents/` 下真实子目录；Rust 递归加载 Markdown

## 约束

- 流程结构更新是显式重建，会替换手工布局和文档无法表达的连线，执行前必须确认节点与连线数量变化
- HTML 块标记在 Obsidian 阅读模式不可见；源码模式可见，删除标记后正文仍可导入，但块 ID、流程角色和叙事单元引用会重新生成
- 旧文档中的 `heading` 场景块继续保留，不做破坏性自动迁移；新场景不再创建该重复层级

## 与现有 AI 助手及 R11-3 的衔接

- AI 改写正文应优先操作 `paragraph`，不得自动把普通正文升级为流程节点
- 叙事分析可读取 `flowRole`，区分纯文稿、节拍和独立节点
- 文档提案必须保持块 ID；改变块类型时应视为结构变更并重新验证关联流程
- Obsidian 重载后的正文以 Markdown body 为准，`syncNarrativeUnits` 再把显式节拍与节点内容传播到流程
- R11-3 已把场景文档纳入收藏与批量分类、状态、模板和归档；批量编辑只作用于场景级元数据，不会隐式改写普通正文块或 `flowRole`

## R11-3 收尾

- 流程、实体、资源、文档、资料共用收藏状态、收藏夹过滤和空状态
- Navigator 的 Ctrl / Shift 多选工具条增加批量编辑，整批修改走单次 `commit`
- 实体支持批量类型与模板；资源支持模板与标签；文档支持分类、状态与模板；资料支持分类、置顶与标签；五类对象都支持收藏与文件夹归档
- 模板批量分配后自动安全补齐缺失字段，不覆盖已有内容
- 实体、资料、文档的收藏状态随 Obsidian Markdown frontmatter 无损往返；旧项目中的非法收藏值在加载时清理
