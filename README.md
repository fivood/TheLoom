# TheLoom 叙事织机

TheLoom 是一款面向写作者与叙事设计师的本地优先创作工具,用于构建分支叙事、管理故事设定与组织写作前期资料。其结构化叙事方法参考 articy:draft,并结合了表格式大纲、故事时间线与资料卡片归档等写作方法论。

支持网页版与 Windows 桌面版(Tauri),界面为黑白灰浅色设计,所有数据默认保存在本地。

## 功能模块

### 流程编辑器

节点式分支叙事编辑器,提供七类节点:对白、剧情片段、汇聚点、条件分支、指令、跳转、出口。节点间拖拽连线构成叙事图,连线可附加标签(如选项文本)。

- **无限嵌套子流程**:剧情片段节点内含独立子画布,双击进入,层级不限,面包屑导航返回;片段节点显示其后代节点总数
- **出口引脚**:子流程内的出口节点自动成为父层片段的命名连接点,子流程可经不同出口接回父层的不同分支
- **演出模式**:从任意节点开始播放流程,对白逐条呈现、分支处提供选项;条件节点依据变量实时求值自动走向,指令节点执行变量赋值;自动进出子流程,附变量状态面板
- **剧本导出**:将流程按线性段落导出为结构化 Markdown 文稿,分支标注选项与去向,子流程递归内嵌,支持导出单个或全部流程

### 实体库

角色、地点、物品、阵营、设定五类实体卡片,支持自定义字段(如动机、恐惧、口头禅)与头像图片(自动裁剪缩放)。角色实体可在流程对白中被引用为说话人。

实体属性面板提供**反向引用**:列出该实体在全项目中的所有出现位置——作为说话人的对白、关联的时间线事件、各模块文本中的名称提及,点击直达。

### 文件夹式 Navigator

流程、实体、资源、文档和资料均可建立多级文件夹。对象可从左侧 Navigator 或属性面板移动到任意文件夹；删除文件夹时内容自动回到“未分组”，不会连带删除正文。类型、分类和标签作为独立筛选条件保留。

### 头脑风暴

自由画布上的便签板。双击空白处新建便签,便签之间可拖拽连线形成关联网络,支持多级灰阶配色。

### 表格大纲

行列式章节大纲:每行一章(章节号、故事时间、标题、主线剧情),每列一条自定义剧情线(伏笔、感情线、配角线等)。逐格检查每条线索在每一章的进展与留白,适用于多线并行的长篇结构规划。

### 时间线

以轨道与时间点构成的故事时间矩阵:轨道对应线索归属(主线、暗线、世界事件),时间点为任意粒度的故事时刻。事件卡可关联实体。时间线记录事件的实际发生顺序,与大纲的叙述顺序互补,适用于倒叙、插叙结构的对照管理。

### 资料卡片

写作前期资料的归档系统:卡片支持分类、多标签、全文搜索、置顶与来源记录,用于沉淀世界观设定、考据材料与灵感来源。

### 全局变量

布尔、数值、文本三类变量,配合流程中的条件分支与指令节点追踪剧情状态,并在演出模式中实时生效。

## 通用能力

- **全局搜索**:Ctrl+K 打开,覆盖全部模块(含任意深度的子流程),结果分组展示,回车直达目标位置——流程结果会打开对应层级并选中节点
- **撤销与重做**:Ctrl+Z / Ctrl+Y,50 步历史,连续输入自动合并为单步
- **项目导入导出**:完整项目可导出为 JSON 文件用于备份与迁移
- **恢复与备份**:保存新内容前自动保留滚动恢复点;损坏存档会被隔离并优先尝试恢复,界面异常时仍可下载紧急备份

## 数据存储

- **网页版**:数据保存在浏览器 localStorage,不上传任何内容
- **桌面版**:可将项目绑定至任意本地文件夹,所有改动自动落盘。文件夹内实体、资料卡和文档以带 YAML frontmatter 的 Markdown 存储,可直接作为 Obsidian 库打开:卡片即笔记,支持双链、标签与图谱;在 Obsidian 中新建或修改的笔记可重新载入。对象的 Navigator 归属以稳定的 `folderId` 保存。写入采用临时文件替换并保留 `project.json.bak`,主文件损坏时自动回退。将文件夹置于 OneDrive、Google Drive 等网盘同步目录即获得云端备份与多设备访问

## AI 智能助手

可选功能。启用后模型只能读取白名单工具与显式选择的对象,输出必须通过本地脚本类型检查、体检与路径测试才能应用,应用是一次 `commit`,可 Ctrl+Z 整体回滚。

- **凭据存储**:网页版 API Key 只存当前浏览器 localStorage;桌面版存 Windows 凭据管理器;绝不写入项目文件、绝不进入云协作同步、绝不出现在诊断导出
- **发送范围**:每次请求前列出对象数、字符数、模块与来源;默认只发当前对象与显式选择的关联;可选整项目但需用户确认
- **绝不擅自修改**:模型输出结构化提案,先在项目副本上执行脚本 / 引用 / 体检 / 路径验证,blocked 提案不显示应用按钮
- **可切换服务商**:OpenAI 兼容(DeepSeek / Kimi / 千问 / GLM / MiniMax / 硅基流动 / 火山方舟等)、Anthropic、Ollama 本地模型;自定义 base URL
- **可取消**:所有长请求都有停止按钮,取消后本地不留残留状态

## 多人协作

基于云端房间的接力式协作:一人以房间码与口令创建房间并推送项目,同伴凭相同凭据拉取。

- **端到端加密**:口令在本地派生鉴权令牌与 AES-256-GCM 加密密钥,项目压缩加密后上传,服务器仅存密文与令牌哈希,无法读取内容;口令遗失则数据不可恢复
- **版本控制**:乐观锁保证推送不会静默覆盖他人改动,版本冲突时可选择拉取云端或强制覆盖
- 网页版与桌面版均可使用;协作后端为可选部署,未启用时不影响其余功能
